1. **What is the effort to onboard NLWeb?**
Because organizations can use the NLWeb Stack to bootstrap (e.g., the app, deployment services, and support services), the main effort will be on enablement, workflow customization, and integration with the organization's downstream services.

1. **What are the steps for a user to onboard NLWeb?**

    1. Install an NLWeb wallet from an app store;
    1. Fill out the Persona with standard data sets;
    1. Add payment methods to the Payer;
    1. Start touching/scanning NFC/QR locators.

1. **What are the steps for a Seller to onboard NLWeb?**

    1. Bootstrap the Seller domain (e.g., nlweb.any-seller.com);
    1. Integrate the Collector with the organization's bank and ERP;
    1. Configure Host workflows integrated with the organization's IT;
    1. Publish a Manifest with Trusts to Brokers and Payers;
    1. Order a batch of physical NFC/QR locators from a Printer;
    1. Deploy NFC/QR locators (e.g., web, app, stores, products).

1. **Can NLWeb be integrated into existing apps?**

    Yes. NLWeb can be added as a chat window to an existing mobile app, allowing the user to interact with the service via text. The chat component must support the minimum feature requirements for chat over the NLWeb protocol.

1. **Do wallets from Google, Apple, and the EU support NLWeb?**

    No. As of 2024, these wallets don't support the NLWeb protocol.
