# ⚙️ Solution Landing Page

Welcome to the **Solution** section of the documentation. This area provides a comprehensive overview of the architecture, user interfaces, data flows, domain models, agent domains, build domains, and edge components that make up the solution.

## 📂 Folder Structure

- **10 Intro/**: Foundational concepts, onboarding, design decisions, and ecosystem overview.
- **20 🧑‍🦰 UI/**: User interface modules, brokers, notifiers, and wallets.
- **25 🔆 Locators/**: Locator systems, robots, tapbands, and things.
- **30 🧩 Data/**: Data models, binds, codes, datasets, files, manifests, messages, tokens, and trusts.
- **35 💬 Chats/**: Chat modules, prompts, scripts, and talkers.
- **37 Scripts 📃/**: Command scripts, functions, holders, and prompts.
- **40 👥 Domains/**: Domain models, flows, and schemas.
- **41 🎭 Domain Roles/**: Roles such as brands, consumers, helpers, payers, sellers, and more.
- **45 🤲 Helper domains/**: Helper modules including advertisers, alarms, authorities, billers, collectors, firewalls, graphs, hosters, keymakers, listeners, loggers, payers, printers, relayers, and wands.
- **50 🫥 Agent domains/**: Agent modules like advisors, concierges, curators, custodians, editors, finders, identifiers, keyboxes, minglers, navigators, payers, personas, reviewers, schedulers, storage, timeline, and vitalogists.
- **55 👷 Build domains/**: Build and sync modules.
- **60 🧰 Edge/**: Edge components including pluggables, ephemerals, palmists, selfies, relayers, scanners, terminals.
- **70 🌳 Ambient/**: Ambient modules such as padlocks and keyholders.
